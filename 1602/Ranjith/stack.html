STACK

  • An Abstract Data Type.
  • Follow "Last In First Out" principle.
  • Data Added or removed only at one end.

Applications

  • Undo in text editors.
  • String Reversal.

Terms

top

  • Pointing the element in top of the stack

push

  • Inserting the element into the stack
  • top++;

pop

  • Removing an element from the stack
  • top--;

Underflow

  • Stack empty.
  • top = -1;

Overflow

  • Stack Full.
  • top = MAX;

Algorithm

push

  1. Check for Overflow
  2. top = top+1
  3. push at top

Algorithm

pop

  1. Check for Underflow
  2. pop at top
  3. top = top-1

QUEUE

  • An Abstract Data Type.
  • Follow "First In First Out" principle.
  • Data Added at one end & removed at one end.

Applications

  • Process Scheduling.
  • IO Buffers.

Terms

Front

  • Pointing the first element added.

Rear

  • Pointing to the last element added

Enqueue

  • Process of Insertion

Dequeue

  • Process of Deletion.

Algorithm - Classical QUEUE

Enqueue

  1. Check for FULL
  2. Rear++
  3. Insert at Rear

Algorithm - Classical QUEUE

Dequeue

  1. Check for EMPTY
  2. Remove from Front
  3. Front++

Disadvantage

The Dequeue'd space can't be reused.

Types

  1. Circular Queue
  2. Double Ended Queue
  3. Priority Queue

Circular Queue

  • a.k.a. Ring Buffer.
  • Reuse of memory.

Operation

Double Ended Queue

  • a.k.a. DEQue.
  • Deletion and Insertion in Both End.
  • Types:
    1. Input Restricted Deque.
    2. Output Restricted Deque.

Priority Queue

  • Insertion asusal.
  • Deleted based in the priority.
    • If same priority
      • First added element is dequeue’d.
