Search Algorithm

  • keywords
    • n : Total number of elements
    • Target : Number to be search
    • PIVOT : Middle position
    • L : Left end
    • R : Right end
  • Algorithm
    1. Set L to 0 and R to n-1
    2. If L > R , number not found
    3. Set PIVOT as (R + L)/2
    4. if PIVOT_val < Target, set L = PIVOT + 1, Go to point 2
    5. if PIVOT_val > Target, set R = PIVOT - 1, Go to point 2
    6. if PIVOT_val = Target, number found

Comparison of linear and binary search

  • linear search scans one item at a time, without jumping to any item.
    • The worst case complexity is O(n), sometimes known an O(n) search
    • Time taken to search elements keep increasing as the number of elements are increased.
  • A binary search however, cut down your search to half as soon as you find middle of a sorted list.
    • The middle element is looked to check if it is greater than or less than the value to be searched.
    • Accordingly, search is done to either half of the given list
